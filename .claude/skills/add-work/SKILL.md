# add-work

Notionの「Works」ページに新しい予定を1件追加するスキル。

- ページURL: https://app.notion.com/p/Works-364f1c78ada280b19e09ec1bdf9361cf
- ページID: `364f1c78ada280b19e09ec1bdf9361cf`（`scripts/generate.py` の `NOTION_WORKS_PAGE_ID` と同一）

このページの内容は本リポジトリのトップページにも `scripts/generate.py` の `fetch_works()` /
`render_works()` で取り込まれて表示される。そのパーサーは「📅 予定一覧」見出し配下の
`bulleted_list_item` をそのまま予定として表示するだけなので、既存の書式を崩さないことが重要。

## 手順

1. 追加する予定の情報を確認する。`args` で渡されていない項目はユーザーに質問する。
   - タイトル（必須）
   - 日付（必須）
   - 時刻（任意。指定がなければ「終日」とする）
   - 補足情報（任意。複数行になる場合は改行ではなく `<br>` で区切る）

2. `mcp__Notion__notion-fetch` でページID `364f1c78ada280b19e09ec1bdf9361cf` を取得し、
   「📅 予定一覧」セクションの現在の内容（一番下の予定の行と、その直後の `---` 区切り線）を確認する。

3. 既存の書式に揃えて新しい行を作成する。
   ```
   - **タイトル** — YYYY年M月D日（曜）HH:MM〜
   ```
   - 曜日は日付から計算して括弧で入れる（例: 2026年7月1日（水））。
   - 時刻指定がない場合は「終日」にする（例: `— 2026年7月1日（水）終日`）。
   - 補足情報がある場合は末尾に `<br>補足1 <br>補足2` の形式で追記する。

4. `mcp__Notion__notion-update-page` を `command: "update_content"` で呼び出し、
   `content_updates` に1件、`old_str` に「📅 予定一覧」セクション内の現在の最後の予定の行、
   `new_str` にその行 + 改行 + 手順3で作った新しい予定行を指定して追記する。
   （`old_str` は手順2で取得した実際の本文と完全一致させること。）

5. 追加後、再度 `mcp__Notion__notion-fetch` でページを取得し、新しい予定が
   「📅 予定一覧」に反映されていることを確認してユーザーに結果を報告する。

## 注意

- 「📋 課題」セクション（to_doブロック）には追加しない。このスキルは予定一覧専用。
- 既存の予定・課題の文言やチェック状態は変更しない。
- Notion MCPツールが利用できない環境では、その旨をユーザーに伝えて中断する。
